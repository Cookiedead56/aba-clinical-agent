# 系统架构

## 概述

ABA Clinical Agent 采用"配置驱动"的 AI Agent 架构，将专业领域知识编码为可执行的 Skill 文件，通过大语言模型作为运行时引擎来执行临床工作流。

## 架构分层

```
┌─────────────────────────────────────────────┐
│  用户交互层                                    │
│  Claude Code CLI / Cursor / Cline            │
├─────────────────────────────────────────────┤
│  角色定义层  →  CLAUDE.md                      │
│  系统的"宪法"：专业边界、绝对红线、执行范式       │
├─────────────────────────────────────────────┤
│  配置层  →  _config.md                        │
│  目录规范、命名规则、frontmatter标准、操作协议    │
├─────────────────────────────────────────────┤
│  路由层  →  _router.md                        │
│  关键词→Skill 映射、易混淆场景澄清、工作流链路    │
├─────────────────────────────────────────────┤
│  技能层  →  29个 SKILL.md                     │
│  每个 Skill 定义：安全协议/输入/输出/执行步骤     │
├─────────────────────────────────────────────┤
│  知识层  →  references/                       │
│  VB-MAPP 领域、prompt hierarchy、胜任力矩阵、发展序列     │
├─────────────────────────────────────────────┤
│  数据层  →  Obsidian-Vault/                   │
│  8层标准目录结构，Markdown + frontmatter + wikilink  │
└─────────────────────────────────────────────┘
```

## Skill 调度流程

1. 用户输入自然语言指令
2. LLM 读取 `_router.md`，匹配关键词到目标 Skill
3. LLM 读取 `_config.md` 获取全局规范
4. LLM 读取目标 `SKILL.md` 获取执行步骤
5. LLM 按步骤读取 Vault 数据、检索知识库
6. LLM 生成输出，执行preview confirmation流程
7. 用户确认后，LLM 写入文件并追加change log

## 安全机制

- **数据敬畏**：找不到数据时标注 `[TBD]`，不编造
- **隐私铁律**：不猜测真实姓名，优先执行 `privacy-filter`
- **基建底线**：严格遵守目录树，不在树外创建文件
- **Human-in-the-Loop**：修改Master Profile前必须展示 Diff 预览等待确认
